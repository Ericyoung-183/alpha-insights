# Three Horizons | 三层面增长战略

**创始人/来源**: McKinsey & Company
**核心价值**: 平衡短期业绩与长期增长，系统规划企业的三层增长曲线
**一句话**: 企业必须同时管理三个时间维度的增长——今天的核心业务、明天的成长业务、后天的种子业务

---

## 概述

三层面增长（Three Horizons of Growth）框架由麦肯锡公司开发，用于管理多时间维度的增长投资组合。

**核心设计理念**：
- **时间维度**：将增长机会按时间分为三个层面
- **组合思维**：同时管理短期、中期、长期增长引擎
- **动态平衡**：避免过度关注短期而忽视长期

**最佳使用时机**：
- 多时间维度增长规划
- 创新投资组合管理
- 平衡短期业绩与长期布局
- 战略节奏设计

**输出价值**：
- 三层面机会清单（H1/H2/H3）
- 各层面资源配置建议
- 增长接力路径图
- 关键里程碑设计

## 一、框架概述

### 1.1 三层面增长模型

```
收入/利润
   ↑
   │
   │    H3: 未来业务
   │    (5-10 年)
   │     ╱
   │    ╱      H2: 成长业务
   │   ╱       (2-5 年)
   │  ╱       ╱
   │ ╱   H1: 核心业务
   │╱    (1-2 年)
   └──────────────────────────→ 时间

   H1: 核心业务 - 现金流来源
   H2: 成长业务 - 增长引擎
   H3: 种子业务 - 未来机会
```

### 1.2 三层业务的特征对比

| 维度 | H1 核心业务 | H2 成长业务 | H3 种子业务 |
|------|-----------|-----------|-----------|
| 时间跨度 | 1-2 年 | 2-5 年 | 5-10 年 |
| 收入贡献 | 主要来源 | 快速增长 | 探索期 |
| 增长模式 | 效率驱动 | 规模扩张 | 创新驱动 |
| 确定性 | 高 | 中 | 低 |
| 资源投入 | 优化投入 | 扩张投入 | 探索投入 |
| 管理重点 | 运营效率 | 市场扩张 | 模式验证 |
| KPI | 利润、现金流 | 增长率、市占率 | 学习、验证 |

### 1.3 为什么需要三层面？

**企业增长的典型困境**:
```
┌─────────────────────────────────────────────────────┐
│  困境 1: 只关注 H1                                  │
│  ───────────────                                  │
│  • 短期业绩好，但增长见顶                            │
│  • 找不到第二曲线                                   │
│  • 例子：柯达、诺基亚手机                            │
├─────────────────────────────────────────────────────┤
│  困境 2: H2 投入不足                                │
│  ───────────────                                  │
│  • 核心业务下滑，成长业务未成型                      │
│  • 增长断档                                         │
│  • 例子：传统零售转型缓慢                            │
├─────────────────────────────────────────────────────┤
│  困境 3: H3 过度投入                                │
│  ───────────────                                  │
│  • 远期探索消耗过多资源                             │
│  • 短期业绩承压                                     │
│  • 例子：乐视生态                                   │
└─────────────────────────────────────────────────────┘
```

## 二、三层面详细解析

### 2.1 H1 核心业务 (Core Business)

**定义**: 当前的主要收入和利润来源。

**核心目标**: **最大化现金流**，为 H2/H3 输血。

**关键策略**:
| 策略类型 | 具体措施 | 目标 |
|---------|---------|------|
| 效率提升 | 流程优化、自动化 | 降低成本 |
| 市场深耕 | 提升渗透率、复购率 | 延长生命周期 |
| 产品迭代 | 延续性创新 | 维持竞争力 |
| 客户运营 | 提升 LTV、降低流失 | 最大化客户价值 |

**H1 健康度指标**:
```
┌────────────────────────────────────────┐
│           H1 健康度诊断                │
├────────────────────────────────────────┤
│ • 收入增长率 vs 行业平均               │
│ • 利润率趋势                          │
│ • 市场份额变化                        │
│ • 客户满意度/NPS                      │
│ • 现金流状况                          │
└────────────────────────────────────────┘
```

**H1 预警信号**:
- 连续 2 期增长放缓
- 利润率持续下降
- 核心客户流失率上升
- 颠覆性竞争者出现

### 2.2 H2 成长业务 (Growth Business)

**定义**: 已验证商业模式，处于快速扩张期的业务。

**核心目标**: **规模化扩张**，打造第二增长曲线。

**关键策略**:
| 策略类型 | 具体措施 | 目标 |
|---------|---------|------|
| 市场扩张 | 新区域、新渠道 | 扩大覆盖 |
| 用户增长 | 获客、转化 | 提升规模 |
| 产品扩展 | 品类延伸、场景拓展 | 提升 ARPU |
| 能力建设 | 团队、供应链、技术 | 支撑增长 |

**H2 健康度指标**:
```
┌────────────────────────────────────────┐
│           H2 健康度诊断                │
├────────────────────────────────────────┤
│ • 收入增长率 (目标>50%)                │
│ • 市场份额变化                        │
│ • Unit Economics (LTV/CAC)            │
│ • 增长飞轮效应                        │
│ • 现金流盈亏平衡点                    │
└────────────────────────────────────────┘
```

**H2 关键决策点**:
- **何时加大投入**: 模式验证成功，Unit Economics 转正
- **何时收缩**: 增长放缓且无法找到新增长点
- **何时剥离**: 与核心战略协同度低

### 2.3 H3 种子业务 (Future Business)

**定义**: 探索性业务，商业模式待验证。

**核心目标**: **快速试错学习**，找到未来方向。

**关键策略**:
| 策略类型 | 具体措施 | 目标 |
|---------|---------|------|
| 小步快跑 | MVP 验证、快速迭代 | 降低试错成本 |
| 多元探索 | 多个方向并行 | 分散风险 |
| 开放创新 | 投资、收购、合作 | 借力外部 |
| 人才储备 | 吸引创新人才 | 能力建设 |

**H3 健康度指标**:
```
┌────────────────────────────────────────┐
│           H3 健康度诊断                │
├────────────────────────────────────────┤
│ • 探索项目数量                        │
│ • 验证速度 (假设→验证周期)             │
│ • 学习产出 (认知积累)                  │
│ • 晋升 H2 的项目数                     │
│ • 战略期权价值                        │
└────────────────────────────────────────┘
```

**H3 管理原则**:
- **容忍失败**: 失败率<70% 说明不够激进
- **快速止损**: 验证失败立即转向
- **小团队**: 保持敏捷，避免官僚化
- **独立考核**: 不用 H1 的 KPI 考核 H3

## 三、三层面执行步骤

### Step 1: 业务分类盘点

**目标**: 将现有业务归类到 H1/H2/H3。

**分类标准**:
```
分类决策树:

业务是否已盈利？
├─ 是 → 是否是主要收入来源？
│      ├─ 是 → H1 核心业务
│      └─ 否 → 是否是高增长？
│             ├─ 是 → H2 成长业务
│             └─ 否 → H1 边缘业务 (考虑剥离)
│
└─ 否 → 是否有收入？
       ├─ 是 → H2 早期业务
       └─ 否 → H3 种子业务
```

**盘点输出**:
| 业务名称 | 分类 | 收入 | 增长率 | 利润 | 战略重要性 |
|---------|------|------|--------|------|-----------|
| 业务 A | H1 | ... | ...% | ... | 高/中/低 |
| 业务 B | H2 | ... | ...% | ... | 高/中/低 |
| 业务 C | H3 | ... | ...% | ... | 高/中/低 |

### Step 2: 三层健康度诊断

**目标**: 评估当前三层结构是否健康。

**诊断维度**:
```
┌─────────────────────────────────────────────────────┐
│              三层健康度诊断框架                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. H1 健康度:                                      │
│     • 现金流是否充足？                              │
│     • 增长是否见顶？                                │
│     • 护城河是否稳固？                              │
│                                                     │
│  2. H2 准备度:                                      │

│     • 是否具备扩张条件？                            │
```

### Step 3: 资源分配设计

**目标**: 设计 H1/H2/H3 的资源分配比例。

**分配原则**:
```
典型资源分配 (按企业阶段):

成熟企业:
H1: 70%  (防守核心)
H2: 20%  (扩张成长)
H3: 10%  (探索未来)

增长企业:
H1: 50%
H2: 35%
H3: 15%

创业企业:
H1: 30%
H2: 40%
H3: 30%
```

**资源类型**:
| 资源类型 | H1 分配 | H2 分配 | H3 分配 |
|---------|--------|--------|--------|
| 资金 | 优化投入 | 扩张投入 | 探索投入 |
| 人才 | 精英运营 | 精锐扩张 | 创新人才 |
| 管理层注意力 | 30% | 50% | 20% |
| 技术资源 | 维护优化 | 快速复制 | 前沿探索 |

### Step 4: 层间协同设计

**目标**: 设计三层业务之间的协同机制。

**协同类型**:
| 协同类型 | 说明 | 示例 |
|---------|------|------|
| H1→H2 输血 | H1 现金流支持 H2 | 腾讯游戏支持微信 |
| H1→H2 赋能 | H1 能力支持 H2 | 阿里电商支持菜鸟 |
| H2→H1 反哺 | H2 增长拉动 H1 | 抖音拉动广告 |
| H3→H2 晋升 | H3 验证成功后升入 H2 | 阿里云从 H3 到 H2 |

**协同机制设计**:

```
│                                             │

│  • 探索方向指引                              │
│  • 验证成功晋升通道                          │
│  • 失败项目人才回流                          │
│                                             │
│  H1 ←───→ H3:                               │
│  • 战略方向对齐                              │
│  • 资源间接支持                              │
│                                             │
└─────────────────────────────────────────────┘
```

### Step 5: 动态调整机制

**目标**: 建立三层业务的动态管理机制。

**调整触发条件**:
```
┌─────────────────────────────────────────────┐
│            业务层级调整触发条件              │
```

## 外部研究补充：三层面增长最佳实践

根据麦肯锡研究和实践总结，三层面增长框架的关键要点如下：

### 1. 三层面定义

| 层面 | 时间 | 特征 | 目标 |
|------|------|------|------|
| **H1** | 0-1 年 | 核心业务，现金牛 | 保卫和扩展 |
| **H2** | 1-3 年 | 新兴业务，高增长 | 规模化 |
| **H3** | 3-5 年+ | 期权业务，探索性 | 验证和孵化 |

### 2. 资源配置建议

```
典型配置：
H1: 60-70% 资源（短期业绩）
H2: 20-30% 资源（中期增长）
H3: 5-10% 资源（长期布局）
```

### 3. 不同层面的管理方式

| 层面 | 考核指标 | 管理方式 | 团队类型 |
|------|---------|---------|---------|
| H1 | 收入/利润/份额 | OKR/KPI | 成熟业务团队 |
| H2 | 增长率/单位经济 | 里程碑 | 创业型团队 |
| H3 | 学习进展/验证 | 假设验证 | 探索型团队 |

### 4. 常见错误

| 错误 | 表现 | 修正 |
|------|------|------|
| H1 过度投入 | 忽视 H2/H3，长期增长乏力 | 强制配置 H2/H3 资源 |
| H3 过早考核 | 用 H1 指标考核 H3 | 用学习指标代替业绩指标 |
| 缺乏接力 | 三层之间没有衔接 | 设计 H3→H2→H1 的晋升路径 |

## 实战案例：字节跳动三层面增长分析（2024）

**背景**：字节跳动 2024 年营收预估超 1200 亿美元，是全球增长最快的科技巨头之一。用 Three Horizons 框架拆解其增长引擎组合：

### H1 核心业务：抖音广告 + 电商

| 指标 | 数据 |
|------|------|
| 国内广告 + 电商营收 | >5000 亿元 |
| 增速 | ~30% YoY |
| 利润率 | 高（广告毛利率 >60%） |
| 战略角色 | 现金发动机，为 H2/H3 提供充足弹药 |

抖音的广告 + 电商闭环已进入成熟收割期，是字节最强的 H1 业务。高利润、高现金流，典型的"现金牛"。

### H2 成长业务：TikTok + 飞书

| 业务 | 规模 | 增长性 | 关键风险 |
|------|------|--------|---------|
| TikTok | MAU >15 亿，全球第一梯队 | 商业化加速，电商 GMV 高速增长 | 美国政策风险（被迫出售/禁令） |
| 飞书 | 企业协作市场份额提升 | 仍处于亏损期 | 面临钉钉、企微的激烈竞争 |

TikTok 是最具想象力的 H2，但地缘政治风险使其从"确定性增长"变成"不确定性赌注"。飞书则是典型的 H2 早期业务——模式验证中但尚未盈利。

### H3 探索业务：AI 大模型 + 游戏

| 业务 | 进展 | 验证状态 |
|------|------|---------|
| 豆包（AI 大模型） | 日活破亿，增长迅猛 | 商业模式未验证，仍在烧钱阶段 |
| 游戏 | 投入数百亿，多次调整方向 | 尚无国民级爆款，ROI 存疑 |

AI 大模型是字节当前最激进的 H3 押注，用低价策略快速获客，但变现路径仍不清晰。游戏则是一个"投了很多但没跑通"的 H3，可能面临止损决策。

### 三层健康度评估

| 维度 | 评分 | 判断依据 |
|------|------|---------|
| H1 健康度 | 9/10 | 抖音广告+电商护城河深，现金流极强 |
| H2 准备度 | 6/10 | TikTok 业务本身优秀，但政策风险严重拉低确定性 |
| H3 储备度 | 7/10 | AI 方向正确且执行力强，但游戏方向验证不足 |

### So What

- **核心优势**：字节的 H1 极强（利润丰厚、增长仍在），为 H2/H3 提供充足资金弹药，这是绝大多数企业不具备的奢侈条件
- **最大风险**：H2 面临的地缘政治风险是三层面框架中罕见的"非市场因素"——TikTok 一旦被迫剥离，字节将失去最大的 H2 引擎，增长接力出现断档
- **战略含义**：字节需要加速 H3→H2 的晋升速度（尤其是 AI），作为 TikTok 风险的对冲方案

## 五、常见错误

| 错误类型 | 表现 | 修正方法 |
|---------|------|---------|
| H1 过度投入 | 忽视 H2/H3 建设 | 强制分配资源给 H2/H3 |
| H2 投入不足 | 只给 H1 输血 | 设立 H2 独立预算 |
| H3 过度发散 | 探索方向太多 | 聚焦 3-5 个战略方向 |
| 层间割裂 | 三层各自为战 | 建立协同机制 |
| 考核错配 | 用 H1KPI 考核 H3 | 差异化考核 |
| 缺乏动态调整 | 一成不变 | 定期复盘调整 |

## 七、中国本土化考量

| 维度 | 中国特色 | 案例 |
|------|---------|------|
| 速度优先 | H2/H3 推进速度远超欧美，5 年可完成一个周期 | 拼多多从创立到上市仅 3 年 |
