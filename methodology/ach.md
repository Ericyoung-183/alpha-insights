# ACH 竞争假设分析法

> 核心价值：系统克服证实偏差，提高判断准确性

---

## 定义

ACH（Analysis of Competing Hypotheses）是一种**系统验证多个竞争假设**的方法：

1. 列出**所有可能的假设**（包括你不同意的）
2. 列出**所有证据**
3. 构建"假设 × 证据"矩阵
4. 评估每个证据对各假设的支持/否定程度
5. **证据最不支持的假设，反而可能是正确的**

---

## 使用时机

**典型应用场景**：
- 情报分析和竞争情报
- 多假设评估
- 高不确定性决策
- 战略预判

**使用原则**：
- 列出所有合理假设
- 用证据矩阵系统评估
- 证伪优于证实

**使用流程**：
1. 列出假设（所有可能性）
2. 收集证据（支持和反对）
3. 构建矩阵（证据 vs 假设）
4. 评估一致性（证伪强证据）
5. 形成判断（最可能的假设）

---## 为什么重要

传统思维的陷阱：
- 先有一个倾向性假设
- 只找支持这个假设的证据
- 忽略或淡化反面证据
- 得出偏颇的结论

ACH 通过**强制考虑所有可能性**，系统性地克服证实偏差。

---

## 触发条件

满足以下 **2 条以上** 时启用：

| 条件 | 判断标准 |
|------|---------|
| 结论关键性 | 判断直接影响重大决策（进入/不进入、投/不投） |
| 假设多样性 | 存在 3 个以上合理的、相互竞争的假设 |
| 偏见风险高 | 分析者或用户有明显倾向性 |
| 证据复杂 | 证据之间存在矛盾，或来源多样 |

## 操作步骤

### Step 1：列出所有可能的假设

基于已有信息，列出**所有合理的假设**，包括你不同意的。

标准：
- 每个假设都应该是可能的（不是明显错误）
- 假设之间相互竞争（只能有一个正确）
- 通常 3-6 个假设

例子：
```
问题：竞对 A 为什么增长快？

假设：
H1：补贴策略激进
H2：产品体验更好
H3：渠道拓展成功
H4：品牌效应积累
```

### Step 2：列出所有证据

收集与问题相关的**所有证据**，包括：
- 支持某个假设的证据
- 反对某个假设的证据
- 中性的、与多个假设相关的证据

例子：
```
E1：竞对 A 营销费用同比增长 200%
E2：竞对 A 用户满意度 85%，高于行业平均
E3：竞对 A 在抖音、小红书投放量第一
E4：竞对 A 停止补贴后，用户增速下降 30%
E5：竞对 A 产品迭代速度是行业平均的 2 倍
E6：竞对 A 的 NPS 分数持续上升
```

### Step 3：构建证据矩阵

构建"假设 × 证据"矩阵，评估每个证据对每个假设的影响。

符号：
- `+` 支持
- `++` 强支持
- `-` 反对
- `--` 强反对
- `N` 中性/无关
- `?` 不确定

例子：

| 证据 | H1:补贴 | H2:产品 | H3:渠道 | H4:品牌 |
|------|---------|---------|---------|---------|
| E1:营销费用+200% | + | N | + | + |
| E2:满意度85% | N | ++ | N | + |
| E3:抖音投放第一 | + | N | ++ | + |
| E4:停补贴后降30% | ++ | - | - | - |
| E5:迭代速度快 | N | ++ | N | + |
| E6:NPS上升 | N | + | N | ++ |

### Step 4：分析矩阵

**关键原则：关注证据的"区分度"，而非数量**

- 有些证据对所有假设都支持 → 区分度低，价值低
- 有些证据只支持/反对特定假设 → 区分度高，价值高

上例分析：
- E2、E5 强支持 H2（产品好）
- E4 强支持 H1（补贴是主因），同时反对 H2、H3、H4
- E3 支持 H3（渠道），但也支持 H1、H4

### Step 5：得出结论

**不是看哪个假设"支持最多"，而是看哪个假设"最没有被推翻"**

计算方法：
1. 统计每个假设的 `+`、`++`、`-`、`--` 数量
2. 重点关注 `--`（强反对）的证据
3. 被 `--` 证据反对的假设，可能性大幅降低

上例结论：
- H1（补贴）：有 E1、E3、E4 支持，无强反对 → 可能性高
- H2（产品）：有 E2、E5、E6 支持，但 E4 反对 → 可能性中
- H3（渠道）：有 E3 支持，但 E4 反对 → 可能性低
- H4（品牌）：有 E1、E6 支持，但 E4 反对 → 可能性低

**综合判断**：竞对增长的主要原因是补贴策略，产品体验是次要因素。

### Step 6：敏感性分析

问：如果某个证据是错的，结论会改变吗？

- 关键证据：E4（停补贴后降速）——如果这个证据有误，结论可能反转
- 建议：进一步验证 E4 的可靠性

## 输出格式

```markdown
## ACH 分析结果

### 问题
[分析的问题]

### 假设清单
| 编号 | 假设 |
|------|------|
| H1 | [假设1] |
| H2 | [假设2] |
| H3 | [假设3] |

### 证据矩阵
| 证据 | H1 | H2 | H3 |
|------|----|----|----|
| E1 | + | N | - |
| E2 | ++ | - | N |
| E3 | - | ++ | + |

### 分析结论
- 最可能正确的假设：H1
- 理由：[说明]
- 关键证据：[列出区分度高的证据]
- 敏感性：[哪些证据如果错误会影响结论]

### 置信度
[高/中/低]
```

## 常见错误

### 错误 1：假设不完整

❌ 只列出自己认同的假设
✅ 列出所有合理的假设，包括不同意的

### 错误 2：证据有偏见

❌ 只收集支持某个假设的证据
✅ 收集所有相关证据，包括中性和反对的

### 错误 3：只看支持数量

❌ 哪个假设 `+` 最多就选哪个
✅ 关注 `--`（强反对），排除被推翻的假设

### 错误 4：忽略敏感性

❌ 不检验关键证据的可靠性
✅ 识别关键证据，进一步验证

### 与假设驱动法的关系

- **假设驱动法**：整体工作方式
- **ACH**：关键判断时的强化验证工具

ACH 是假设驱动法的"升级版"，用于更关键、更复杂的判断。
